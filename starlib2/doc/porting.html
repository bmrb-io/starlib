"starlib" porting notes.

The starlib was first developed on an SGI Irix 6.2 machine. Due to some strange things that that platform does, and due to the fact that the C++ STL standard was still in the process of being ratified when it was created, there is a portation problem that will come up when you try to move this to a different platform:

  • SGI's 'CC' compiler did not implement the C++ "string" class in the normal location. With SGI, if you #include the file "<string.h>", you end up with the old-fashioned Ansi-C string header file (the one with strcat(), strcmp(), strlen(), etc...) The new C++ standard says that it should give you the C++ string class, but for whatever reasons, SGI didn't want to do it that way. Their C++ string class is in a file called "mstring.h". Knowing that this would be a problem, I created a seperate file called portatbility.h, which contains the include of mstring.h. To fix this for other non-SGI platforms, simply edit portability.h, and change the string include to whatever is appropriate to your system. In future, SGI claims to have a better, more compliant C++ library with their compiler. When this happens, I will be able to fix this annoyance.
